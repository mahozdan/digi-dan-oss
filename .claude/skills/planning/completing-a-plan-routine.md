## Checklist for completing a plan 

- [ ] Add a new xxx.implementation-summary.md file next to all other plan files 
- [ ] Update .env.example if it's required
- [ ] Update the ./readme.md file if it's required 
- [ ] Split the xxx.user-stories.md and xxx.acceptance-criteria.md files into user-stories.md and acceptance-criteria.md files embeded in the modules where their logic has been implemented
- [ ] Update other user-stories.md and acceptance-criteria.md files if required 
- [ ] Move xxx.plan.md, xxx.specs.md, xxx.md, xxx.user-stories.md, xxx.acceptance-criteria.md, xxx.test-plan.md and xxx.implementation-plan.md, xxx.plan-review.md, xxx.implmentatio-summary.md into ./docs/featues/complete folder
- [ ] Consider adding new tools to the mcp server (when not sure ask the user)
- [ ] Make sure to run npm run dod to verify you met the definition of done
- [ ] Suggest a commit message after reviewing git
